# Техническое задание на проект "Демонстрационный стенд библиотеки Marshmallow"

## 1. Общая информация

**Название проекта:** Демонстрационный стенд библиотеки Marshmallow  
**Заказчик:** Учебная организация HERNY  
**Исполнитель:** Группа HERNY  
**Цель проекта:** Создать консольное приложение для наглядной демонстрации возможностей библиотеки `marshmallow` по сериализации, десериализации и валидации данных.

## 2. Описание задачи (что нужно сделать)

Необходимо разработать модульное консольное приложение, которое в интерактивном режиме демонстрирует практическое применение ключевых инструментов библиотеки `marshmallow`. Приложение должно показывать, как преобразовывать Python-объекты в JSON и обратно, валидировать входящие данные, обрабатывать вложенные структуры и создавать кастомные поля.

## 3. Функциональные требования (что система должна уметь)

1. Система должна предоставлять текстовое меню для выбора демонстрации.
2. Система должна демонстрировать базовую сериализацию Python-объектов в словарь/JSON.
3. Система должна демонстрировать десериализацию и валидацию входящих данных.
4. Система должна показывать обработку ошибок валидации с понятными сообщениями.
5. Система должна демонстрировать работу с вложенными объектами (`fields.Nested`).
6. Система должна демонстрировать создание кастомных полей с собственной логикой валидации.
7. Система должна показывать использование хуков (`@pre_dump`, `@post_dump`) для модификации данных.
8. Система должна демонстрировать частичное обновление (параметр `partial=True`).
9. Система должна корректно обрабатывать неверный ввод пользователя.

## 4. Нефункциональные требования (ограничения)

1. Язык программирования: Python 3.8+
2. Используемые библиотеки: marshmallow (последняя стабильная версия)
3. Интерфейс: консольный (CLI)
4. Архитектура: модульная (разделение на core, utils, logic, presentation)
5. Код должен соответствовать PEP 8
6. Проект должен запускаться на Windows, Linux и macOS

## 5. Требования к интерфейсу

1. При запуске программы отображается главное меню с нумерованными пунктами.
2. Меню содержит следующие пункты:
   - Базовая сериализация (объект → JSON)
   - Валидация данных (JSON → объект)
   - Вложенные структуры (автор, статьи, комментарии)
   - Кастомные поля и хуки
   - Частичное обновление (partial)
   - Полная демонстрация блога
   - Выход
3. Пользователь выбирает пункт, вводя цифру от 0 до 6.
4. После выполнения демонстрации программа возвращается в главное меню.
5. Для выхода из программы используется пункт "0".

## 6. Входные и выходные данные

**Входные данные:**
- Цифра от 0 до 6, введенная пользователем для выбора пункта меню
- Для демонстрации валидации — тестовые JSON-данные (встроенные в код)

**Выходные данные:**
- Результаты сериализации в формате JSON (с форматированием)
- Результаты валидации (успех или подробные сообщения об ошибках)
- Информация о преобразованиях данных на различных этапах
- Статистика и метаданные обработанных объектов

7. Продемонстрирована разница между dump_only и обычными полями.

# Документация проекта "Демонстрационный стенд Marshmallow"

## Структура проекта

```
marshmallow-demo/
├── core.py          # Модели данных и схемы Marshmallow
├── utils.py         # Утилиты, кастомные поля, декораторы
├── logic.py         # Логика демонстраций
├── app.py           # Главное приложение с меню
```

## Описание модулей

### 1. `core.py` - Ядро системы

Содержит модели данных и соответствующие схемы Marshmallow:

**Модели данных:**
- **`User`** - модель пользователя (id, username, email, registered_on, is_active)
- **`Article`** - модель статьи (id, title, content, author, created_at, tags, views)
- **`Comment`** - модель комментария (id, text, author, article, created_at)

**Схемы Marshmallow:**
- **`UserSchema`** - сериализация/валидация пользователя с валидацией email и длины username
- **`ArticleSchema`** - сериализация статьи с вложенным автором и автоматическим созданием превью
- **`CommentSchema`** - сериализация комментария с ограниченной информацией об авторе

### 2. `utils.py` - Утилиты и кастомные поля

Содержит вспомогательные функции и кастомные поля:

- **`pretty_print(data, title=None)`** - форматированный вывод JSON с поддержкой datetime
- **`print_header(text)`** - печать заголовка с разделителями
- **`timer(func)`** - декоратор для замера времени выполнения с сохранением метаданных через `@wraps`
- **`PasswordField`** - кастомное поле для валидации паролей (длина, цифры, регистр)
- **`PhoneField`** - кастомное поле для валидации российских телефонных номеров
- **`validate_password_strength(password)`** - функция проверки сложности пароля
- **`create_sample_registration_data()`** - создание тестовых данных для демо-валидации

### 3. `logic.py` - Логика демонстраций

Содержит функции-демонстрации для каждого пункта меню:

- **`demo_basic_serialization()`** - базовая сериализация объектов в JSON
- **`demo_validation()`** - валидация входящих данных и обработка ошибок
- **`demo_nested_structures()`** - работа со вложенными объектами
- **`demo_custom_fields()`** - демонстрация кастомных полей и хуков
- **`demo_partial_update()`** - частичное обновление объектов
- **`demo_full_blog()`** - комплексная демонстрация всех возможностей

### 4. `app.py` - Главное приложение

Содержит:
- **`show_menu()`** - функция отображения главного меню
- **`main()`** - главный цикл программы с обработкой выбора пользователя
- Обработка исключений и корректный выход из программы

## Демонстрации

### 1. Базовая сериализация (объект → JSON)
Показывает преобразование Python-объектов в JSON:
- Сериализация одного пользователя
- Сериализация списка пользователей
- Демонстрация dump_only полей (id, registered_on)
- Сериализация статьи с вложенным автором

### 2. Валидация данных (JSON → объект)
Демонстрирует проверку входящих данных:
- Валидация корректных данных
- Обработка ошибок с понятными сообщениями
- Проверка email и длины строк

### 3. Вложенные структуры
Показывает работу со связанными объектами:
- Статья с автором и комментариями
- Выборка только определенных полей (only parameter)
- Исключение полей (exclude parameter)

### 4. Кастомные поля и хуки
Демонстрирует расширение возможностей Marshmallow:
- Кастомное поле `PasswordField` с проверкой сложности пароля
- Кастомное поле `PhoneField` для валидации телефонных номеров
- Хуки `@pre_dump` и `@post_dump` для модификации данных

### 5. Частичное обновление (partial)
Показывает обновление только указанных полей:
- Обычная загрузка требует все обязательные поля
- Использование параметра `partial=True` для обновления только username
- Сохранение остальных полей без изменений

### 6. Полная демонстрация блога
Комплексная демонстрация всех возможностей:
- Создание пользователя через валидацию
- Создание статьи
- Добавление комментариев
- Сериализация полной структуры с вложенными объектами
- Статистика созданных объектов

## Ключевые концепции

### 1. Сериализация (dump)
- **Проблема:** нужно отправить Python-объект клиенту в JSON
- **Решение:** `schema.dump(obj)` преобразует объект в словарь
- **Пример:** `UserSchema().dump(user)` → словарь с данными пользователя

### 2. Десериализация и валидация (load)
- **Проблема:** нужно проверить и преобразовать JSON от клиента
- **Решение:** `schema.load(data)` валидирует и возвращает объект/словарь
- **Пример:** `UserSchema().load({'username': 'ivan', 'email': 'ivan@mail.ru'})`

### 3. dump_only поля
- **Проблема:** клиент не должен менять некоторые поля (id, дата создания)
- **Решение:** `dump_only=True` - поле только для вывода
- **Пример:** `id = fields.Int(dump_only=True)`

### 4. Вложенные объекты (Nested)
- **Проблема:** объекты содержат другие объекты
- **Решение:** `fields.Nested(OtherSchema)` для вложенной сериализации
- **Пример:** `author = fields.Nested(UserSchema)`

### 5. Кастомная валидация
- **Проблема:** стандартные валидаторы не покрывают все случаи
- **Решение:** создание своих полей с переопределением `_deserialize`
- **Пример:** `PasswordField` проверяет наличие цифр, длину и регистр

### 6. Хуки (pre_dump, post_dump)
- **Проблема:** нужно модифицировать данные до/после обработки
- **Решение:** декораторы для вмешательства в процесс
- **Пример:** `@pre_dump` увеличивает счетчик просмотров, `@post_dump` добавляет метаданные

## Заключение

Данный проект служит наглядным пособием для понимания практической пользы библиотеки `marshmallow`. Каждая демонстрация решает конкретную проблему разработки API и работы с данными, показывая, как Marshmallow может улучшить надежность, безопасность и поддерживаемость кода.

Библиотека Marshmallow особенно полезна для:
- **Веб-разработки** - валидация входящих запросов и форматирование ответов
- **API-интеграций** - преобразование данных между различными форматами
- **Работы с формами** - проверка пользовательского ввода
- **Документирования данных** - схемы служат спецификацией структуры данных

Проект демонстрирует, как с помощью минимального количества кода можно создать надежную систему валидации и сериализации, которая будет работать без ошибок и обеспечивать целостность данных.
