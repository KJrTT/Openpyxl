# Openpyxl

Openpyxl — это мощная библиотека Python для чтения, записи и редактирования файлов Excel 2010 (`xlsx`/`xlsm`/`xltx`/`xltm`). Она позволяет взаимодействовать с электронными таблицами без необходимости запуска приложения Microsoft Excel, предоставляя разработчикам полный контроль над данными, стилями, формулами и диаграммами. Openpyxl незаменима для автоматизации создания отчетов, обработки больших объемов табличных данных и интеграции Excel-файлов в бизнес-процессы .

## Модуль 1. Контент и предпосылки

### Ограничения стандартных средств + Описание проблемы, которые существуют без данного инструмента.

### Проблема номер 1:
При работе с Excel-файлами вручную или через CSV приходится сталкиваться с неудобствами. Стандартные библиотеки Python (`csv`) не поддерживают форматы с расширенным форматированием, множеством листов, формулами и макросами (`xlsx`). Любая попытка создать сложный структурированный отчет с объединенными ячейками, цветами и диаграммами средствами чистого Python превращается в невероятно сложную задачу по генерации XML-структур вручную.

### Проблема номер 2:
Чтение данных из Excel вручную также проблематично. Без специальной библиотеки вам пришлось бы парсить сложный XML-архив, которым на самом деле является `.xlsx` файл. Это требует глубоких знаний структуры Office Open XML и приводит к написанию огромного объема хрупкого кода для извлечения даже простых значений.

### Проблема номер 3:
Excel-файлы — это не просто сетка с данными. Они содержат метаданные: стили ячеек (шрифты, цвета, границы), объединенные диапазоны, примечания, изображения, диаграммы и сводные таблицы. Поддержка всего этого многообразия без специализированного инструмента — практически невыполнимая задача.

### Проблема номер 4:
Файлы Excel большого размера потребляют много оперативной памяти. Если вы попытаетесь прочитать 100-мегабайтный файл, просто загрузив его в память как XML-дерево, приложение почти наверняка упадет с ошибкой `MemoryError`. Необходим инструмент, умеющий работать с данными потоково, не загружая весь файл целиком.

### Проблема номер 5:
Excel имеет собственное представление о датах (количество дней от 1900-01-00) и формулах. Преобразование этих внутренних форматов в понятные Python-объекты (`datetime`) и обратно требует учета множества нюансов (например, баг с високосным 1900 годом в Excel). Без библиотеки легко ошибиться в расчетах .

## 2. Решения, которые предоставляет библиотека
### Решение проблемы номер 1:
Openpyxl предлагает объектную модель для представления Excel-файла. Вы работаете с классами `Workbook` (книга), `Worksheet` (лист) и `Cell` (ячейка), а библиотека сама сериализует это в корректный XML. Создание структурированного файла становится интуитивно понятным.

```python
from openpyxl import Workbook

# Создаем новую книгу
wb = Workbook()
# Получаем активный лист
ws = wb.active

# Работаем с ячейками как со словарем или через метод
ws['A1'] = "Привет, Мир!"
ws.append([1, 2, 3]) # Добавляем строку

# Сохраняем файл
wb.save("простой_отчет.xlsx")
```
**Принцип работы:** Библиотека инкапсулирует всю сложность формирования XML-структур. Вы манипулируете Python-объектами, а Openpyxl на выходе создает корректный ZIP-архив с XML-файлами, который Excel понимает как книгу .

### Решение проблемы номер 2:
Openpyxl умеет парсить Excel-файл и предоставлять удобный доступ к данным. Вы просто загружаете файл и обращаетесь к нужным листам и ячейкам, получая уже преобразованные типы данных (числа, строки, даты).

```python
from openpyxl import load_workbook

# Загружаем существующий файл
wb = load_workbook('отчет.xlsx')
ws = wb.active

# Читаем значение конкретной ячейки
значение = ws['A1'].value
print(f"Значение A1: {значение}")

# Итерируемся по строкам
for row in ws.iter_rows(min_row=1, max_col=2, values_only=True):
    print(row) # Получаем кортеж со значениями ячеек строки
```
**Принцип работы:** Функция `load_workbook()` распаковывает `.xlsx` файл в памяти, парсит XML-структуры и создает дерево Python-объектов (`Workbook` -> `Worksheet` -> `Cell`), предоставляя удобный интерфейс для доступа к данным и их модификации .

### Решение проблемы номер 3:
Библиотека предоставляет модули для работы со всеми аспектами Excel. Вы можете управлять стилями (шрифт, цвет, заливка, границы), добавлять комментарии, создавать диаграммы, настраивать условное форматирование и фильтры.

```python
from openpyxl import Workbook
from openpyxl.styles import Font, PatternFill
from openpyxl.comments import Comment

wb = Workbook()
ws = wb.active

# Применяем стили
cell = ws['A1']
cell.value = "Важно"
cell.font = Font(bold=True, color="FF0000") # Красный жирный
cell.fill = PatternFill(start_color="FFFF00", end_color="FFFF00", fill_type="solid") # Желтый фон

# Добавляем комментарий
cell.comment = Comment("Это важная ячейка", "Автор")

wb.save("стилизованный_файл.xlsx")
```
**Принцип работы:** Openpyxl включает в себя десятки классов, соответствующих элементам форматирования Excel. Присваивая экземпляры этих классов свойствам объекта `Cell`, вы управляете внешним видом и дополнительными атрибутами, которые библиотека корректно запишет в файл .

### Решение проблемы номер 4:
Openpyxl поддерживает два оптимизированных режима: `read_only` для потокового чтения огромных файлов и `write_only` для потоковой записи без удержания всей структуры в памяти.

```python
from openpyxl import load_workbook
from openpyxl import Workbook

# Потоковое чтение (экономия памяти)
wb_read = load_workbook('огромный_файл.xlsx', read_only=True)
ws = wb_read.active
for row in ws.iter_rows(): # Данные подгружаются по мере итерации
    for cell in row:
        print(cell.value)
wb_read.close()

# Потоковая запись (для огромных данных)
wb_write = Workbook(write_only=True)
ws = wb_write.create_sheet()
for i in range(1, 1000000): # Запись миллиона строк
    ws.append([i, i*2, f"Строка {i}"])
wb_write.save('огромный_результат.xlsx')
```
**Принцип работы:** В режиме `read_only` библиотека не строит полное дерево объектов в памяти, а читает XML-файл с листом построчно. В режиме `write_only` данные сразу оптимизированно записываются в архив на диск, минуя хранение всей книги в оперативной памяти .

### Решение проблемы номер 5:
Openpyxl автоматически конвертирует даты Excel в объекты `datetime` Python при чтении и обратно при записи. Также она правильно обрабатывает формулы, позволяя как записывать их, так и получать результаты вычислений, если файл был открыт в Excel.

```python
from openpyxl import Workbook
from datetime import datetime

wb = Workbook()
ws = wb.active

# Запись даты (автоматически преобразуется в число Excel)
ws['A1'] = datetime.now()

# Запись формулы
ws['A2'] = "=SUM(1, 2)"
# При чтении можно получить либо формулу, либо ее результат
print(ws['A2'].value) # Выведет: '=SUM(1, 2)'
# Если открыть файл data_only=True, то выведет результат вычисления
```

**Принцип работы:** Библиотека знает о внутреннем представлении дат в Excel и использует вспомогательные модули (например, `jdcal`) для конвертации. Для формул она сохраняет строку формулы в нужный XML-тег, а при загрузке с флагом `data_only=True` пытается извлечь кэшированное значение, если файл был ранее сохранен с результатами .

## 3. Инженерная/визуальная грамотность

В контексте Openpyxl инженерная грамотность — это умение выйти за рамки простого "чтения и записи ячеек" и спроектировать эффективный, поддерживаемый и ресурсо-эффективный слой работы с табличными данными.

**Императивный код против декларативного шаблона (Визуальная чистота кода):** Вместо того чтобы писать длинные императивные скрипты, перебирающие ячейки и применяющие стили в куче циклов, грамотный инженер создает шаблоны или использует структуры данных. Он может определить словарь со стилями по умолчанию или использовать `named_styles`, чтобы сделать код более читаемым и централизованно управлять форматированием. Визуально код становится не просто последовательностью команд, а описанием того, *как должен выглядеть итоговый документ*.

**Разделение ответственности (Separation of Concerns):** Грамотный подход требует четкого разделения: слой извлечения/подготовки данных (например, из SQL-запроса или API) и слой представления этих данных в Excel (слой Openpyxl). Нельзя смешивать логику бизнес-расчетов с кодом, устанавливающим ширину столбца или цвет заливки. Инженерная культура проявляется в создании отдельных функций или классов, которые принимают готовые данные и структуру и на их основе генерируют файл, делая код переиспользуемым для разных отчетов.

**Ресурсоэффективность:** Грамотный инженер всегда оценивает объем данных. Он не будет загружать 500-мегабайтный файл в память обычным способом — он сразу спроектирует решение с использованием `read_only` и `write_only` режимов. Это предвидение проблем производительности и нагрузок на инфраструктуру.

## 4. Применение в реальных проектах

### Пример 1: Генерация ежемесячного финансового отчета

**Задача:** Каждый месяц формировать файл Excel с отчетом о продажах, который включает итоговые суммы, форматирование заголовков, автоподбор ширины столбцов и итоговую строку с формулой `SUM`.

**Решение:**
```python
from openpyxl import Workbook
from openpyxl.styles import Font, PatternFill
from openpyxl.utils import get_column_letter

wb = Workbook()
ws = wb.active
ws.title = "Продажи за Март"

# Заголовки
headers = ["Товар", "Количество", "Цена", "Сумма"]
ws.append(headers)

# Данные
data = [
    ["Ноутбук", 5, 75000, "=B2*C2"],
    ["Мышь", 20, 1500, "=B3*C3"],
]
for row in data:
    ws.append(row)

# Формула итого
last_row = ws.max_row
ws[f'D{last_row+1}'] = "=SUM(D2:D{})".format(last_row)

# Стилизация
header_font = Font(bold=True, color="FFFFFF")
header_fill = PatternFill(start_color="4F81BD", end_color="4F81BD", fill_type="solid")
for cell in ws[1]: # Первая строка
    cell.font = header_font
    cell.fill = header_fill

# Автоподбор ширины
for col in ws.columns:
    max_length = 0
    col_letter = get_column_letter(col[0].column)
    for cell in col:
        try:
            max_length = max(max_length, len(str(cell.value)))
        except:
            pass
    ws.column_dimensions[col_letter].width = max_length + 2

wb.save("финансовый_отчет_март.xlsx")
```
**Результат:** Автоматизированный процесс формирования корпоративного отчета, который всегда выглядит единообразно, не требует ручного копирования данных и исключает ошибки при расчетах.

### Пример 2: Обработка загруженного пользователем прайс-листа (Веб-приложение на Flask/Django)

**Задача:** Пользователь загружает прайс-лист в формате Excel. Нужно прочитать данные, провалидировать их (цены должны быть числами) и сохранить в базу данных.

**Решение:**
```python
from openpyxl import load_workbook
from decimal import Decimal

def process_price_list(file_path):
    wb = load_workbook(file_path, data_only=True) # data_only чтобы читать значения, а не формулы
    ws = wb.active

    products = []
    errors = []
    for i, row in enumerate(ws.iter_rows(min_row=2, values_only=True), start=2): # Пропускаем заголовок
        name, price_str = row[0], row[1]
        if not name:
            continue
        try:
            # Валидация: цена должна быть числом
            price = Decimal(str(price_str))
            products.append({"name": name, "price": price})
        except (TypeError, ValueError, InvalidOperation):
            errors.append(f"Строка {i}: Некорректная цена '{price_str}'")
    wb.close()
    return products, errors
```
**Результат:** Надежный и безопасный парсинг пользовательских данных. Флаг `data_only=True` гарантирует, что мы получим числовое значение, а не формулу. Обработка ошибок отделяет валидные данные от невалидных.

### Пример 3: Массовое создание персонализированных документов

**Задача:** На основе списка сотрудников (из CSV) создать для каждого отдельный Excel-файл с персональным расчетом зарплаты за месяц, используя общий шаблон.

**Решение:**
```python
from openpyxl import load_workbook
import csv

with open('employees.csv', 'r') as f:
    reader = csv.DictReader(f)
    for row in reader:
        # Загружаем шаблон для каждого сотрудника
        wb = load_workbook('шаблон_зарплаты.xlsx')
        ws = wb.active

        # Заполняем персональные данные
        ws['B2'] = row['name']
        ws['B3'] = row['position']
        ws['B4'] = row['salary']
        # ... формулы в шаблоне посчитают налоги и итог

        # Сохраняем файл с именем сотрудника
        wb.save(f"зарплата_{row['name']}.xlsx")
```
**Результат:** Принцип "шаблонизации" позволяет быстро создавать множество однотипных документов. Менеджер изменяет один файл-шаблон, и это автоматически применяется ко всей рассылке, что исключает ошибки и экономит часы работы.

# Модуль 2. Основные идеи и механизмы
## 1. Центральные объекты и архитектуры

Основным центральным объектом Openpyxl является **Рабочая книга (`Workbook`)**. Она представляет собой весь Excel-файл и служит корневым контейнером для всех остальных элементов. `Workbook` содержит листы, именованные диапазоны, стили, свойства документа и другие глобальные настройки. Создание или загрузка `Workbook` — это точка входа в библиотеку .

Вторым ключевым объектом является **Рабочий лист (`Worksheet`)**. Это двумерная сетка ячеек, организованных в строки и столбцы. Лист отвечает за хранение данных, размеров строк/столбцов, объединенных ячеек, а также за такие функции, как фильтры, условное форматирование и валидация данных. Листы управляются коллекцией внутри рабочей книги.

Третий фундаментальный объект — **Ячейка (`Cell`)**. Это пересечение строки и столбца, которое содержит непосредственное значение. Ячейка хранит не только данные, но и ссылки на стили (шрифт, заливку, границы), комментарий, а также информацию о своем типе (число, строка, дата, формула). Ячейки не создаются по отдельности, а генерируются при обращении к ним через лист .

Архитектурно Openpyxl построена по принципу **объектной модели документа**. Каждый элемент Excel имеет свое представление в виде Python-класса, а отношения между ними (книга содержит листы, лист содержит ячейки) моделируются через вложенные структуры данных (списки, словари). При сохранении файла эти объекты рекурсивно сериализуются в XML, а при чтении — XML десериализуется обратно в объекты .

Важной частью архитектуры является модуль **`styles`**, который реализует пул стилей. В Excel стили хранятся отдельно от ячеек для экономии места. Openpyxl следует этому принципу: вы определяете объекты `Font`, `Fill`, `Border` и присваиваете их ячейкам. Библиотека сама отслеживает, какие стили уже использовались, и создает ссылки на них в итоговом файле, избегая дублирования.

Еще один важный аспект — **система измерений**. Openpyxl использует внутренние единицы Excel (для ширины столбцов) и предоставляет утилиты для их конвертации. Это позволяет точно контролировать размеры элементов на листе, понимая, как они будут выглядеть при открытии в Excel.

Архитектура библиотеки также включает мощный слой **совместимости с типами данных**. Модуль `utils` содержит функции для конвертации между координатами в формате "A1" и индексами (строка, колонка), а также для безопасной обработки строк с недопустимыми в XML символами. Это изолирует разработчика от низкоуровневых проблем формата.

## 2. Ключевые механизмы работы, возможности, принципы использования

### 1) Механизм создания/загрузки рабочей книги
Основной механизм инициализации. Для создания новой книги используется класс `Workbook()`. Для открытия существующей — функция `load_workbook()`. Второй метод предоставляет параметры (`read_only`, `data_only`), определяющие режим работы с файлом.

```python
from openpyxl import Workbook, load_workbook
wb_new = Workbook()
wb_existing = load_workbook('file.xlsx')
```

### 2) Механизм управления листами
Каждая рабочая книга содержит коллекцию листов. Openpyxl позволяет получать активный лист (`wb.active`), создавать новые (`create_sheet`), переименовывать (`ws.title`), получать список всех листов (`wb.sheetnames`) и удалять их (`remove`).

```python
ws = wb.create_sheet("Новый лист", 0) # Вставить в начало
sheet_list = wb.sheetnames
```

### 3) Механизм доступа к ячейкам
Поддерживается два основных способа: через ключ-координаты (как в словаре, `ws['A1']`) и через числовые индексы строки и столбца (`ws.cell(row=1, column=1)`). Второй способ удобен для итерации в циклах.

```python
cell1 = ws['B2']
cell2 = ws.cell(row=2, column=2, value=10) # Создание и запись
```

### 4) Механизм итерации по данным
Для эффективного обхода больших диапазонов используются методы `iter_rows()` и `iter_cols()`. Они позволяют перемещаться по строкам или столбцам в заданных границах и могут возвращать только значения (`values_only=True`), что экономит память .

```python
for row in ws.iter_rows(min_row=1, max_row=5, values_only=True):
    print(row)
```

### 5) Механизм работы с формулами
Формула задается как строка, начинающаяся с `=`. При записи библиотека помечает такой тип данных как формулу. При чтении с параметром `data_only=False` (по умолчанию) возвращается строка формулы. При `data_only=True` возвращается последнее вычисленное значение (если оно было сохранено в файле).

```python
ws['A10'] = "=SUM(A1:A9)"
```

### 6) Механизм стилизации (Styles)
Стилизация реализована через отдельные классы: `Font`, `PatternFill`, `Border`, `Alignment` и др. Экземпляры этих классов создаются и присваиваются свойствам ячейки (`cell.font`, `cell.fill`). Openpyxl использует пул стилей, чтобы минимизировать размер выходного файла .

```python
from openpyxl.styles import Font, PatternFill
cell.font = Font(bold=True, size=14)
cell.fill = PatternFill(start_color="FFC7CE", end_color="FFC7CE", fill_type="solid")
```

### 7) Механизм управления размерами
Ширина столбцов и высота строк настраиваются через словари `column_dimensions` и `row_dimensions` рабочего листа.

```python
ws.column_dimensions['A'].width = 30
ws.row_dimensions[1].height = 40
```

### 8) Механизм потокового ввода/вывода (оптимизированные режимы)
Для работы с большими файлами существуют специальные режимы: `read_only` для итеративного чтения и `write_only` для итеративной записи. В этих режимах библиотека не строит объектную модель всей книги в памяти, что позволяет обрабатывать файлы, размер которых превышает доступную оперативную память .

```python
from openpyxl import load_workbook
wb = load_workbook('big_file.xlsx', read_only=True)
```

### 9) Механизм добавления строк (append)
Предоставляет удобный способ добавления данных в конец листа. Принимает итерируемый объект (список, кортеж) и размещает его элементы последовательно в ячейках текущей строки, автоматически переходя на новую строку.

```python
ws.append(["ID", "Name", "Price"])
ws.append([1, "Item 1", 100.50])
```

### 10) Механизм объединения ячеек
Позволяет объединять несколько ячеек в одну большую для создания заголовков или сложных макетов. Объединенный диапазон становится одной ячейкой с адресом по верхней левой координате.

```python
ws.merge_cells('A1:D1')
ws.unmerge_cells('A1:D1') # Разъединить
```

### 7. Работа со структурированными данными

Openpyxl — это инструмент для преобразования между сырыми данными в Python и строго структурированным форматом Excel. Он превращает плоские и иерархические данные в организованные строки и столбцы.

Для загрузки данных из внешних источников (CSV, JSON, базы данных) идеально подходит механизм `append`. Вы можете подготовить список списков или список словарей и затем в цикле перенести их в Excel. Это позволяет быстро консолидировать информацию из разных источников в одном файле.

При работе с иерархическими данными (например, заказ и список товаров в нем) Openpyxl позволяет визуально структурировать их на листе: можно создать заголовок заказа, объединить ячейки для его полей, а ниже, начиная со следующей строки, построить таблицу с позициями заказа, используя `append`. Это делает итоговый документ читаемым для человека.

Механизм `iter_rows` с параметром `values_only=True` — это основной инструмент для обратной задачи: превращения структурированного листа Excel обратно в структуры данных Python. Вы можете прочитать диапазон и преобразовать его в список словарей, где ключами будут значения из заголовков столбцов. Это позволяет легко интегрировать данные из Excel в аналитические пайплайны или базы данных.

Для работы с многолистовыми структурами (например, книга "Отчет за год" с листами "Январь", "Февраль"...) Openpyxl позволяет обращаться к листам по имени. Это дает возможность организовать данные тематически: например,汇总 данных по филиалам на разных листах, но в одной книге.

---
### 8. Итеративные элементы

**Итерация по строкам и столбцам (iter_rows / iter_cols):** Ключевая итеративная возможность. Позволяет обходить ячейки в заданном прямоугольном диапазоне построчно или постолбцово, экономя ресурсы по сравнению с обращением по каждому адресу.

**Автоматическое преобразование типов (Type Coercion):** Openpyxl автоматически определяет тип записываемых данных. Если передать `datetime`, он сохранится как дата. Число сохранится как число. Строка — как строка. Это избавляет от необходимости вручную выставлять тип ячейки.

**Автоматическое определение размеров:** Хотя Openpyxl не вычисляет "автоподбор ширины" автоматически, он предоставляет все данные для этого. Вы можете в цикле пройти по столбцу, измерить максимальную длину содержимого и установить ширину столбца программно, автоматизируя подгонку.

**Автоинкремент адресов при добавлении (append):** Метод `append()` автоматически определяет следующую пустую строку и записывает данные в нее, что делает заполнение листа похожим на работу с курсором в базе данных.

---
### 9. Обработка и отладка ошибок

Работа с файловой системой и сложным форматом Excel неизбежно приводит к ошибкам. Openpyxl использует иерархию исключений для информирования о проблемах.

**Основные типы исключений:**
*   `openpyxl.utils.exceptions.InvalidFileException`: Возникает при попытке открыть файл, который не является поддерживаемым форматом Excel (например, старый `.xls`) .
*   `openpyxl.utils.exceptions.CellCoordinatesException`: Ошибка при неверном формате координат ячейки.
*   `ValueError` или `TypeError`: Могут возникать при попытке записать в ячейку неподдерживаемый тип данных (например, пользовательский класс без str-представления).

**Отладка формул:** Самая частая проблема — записанная формула не работает в Excel. Для отладки нужно убедиться, что библиотека не добавляет лишних пробелов и что разделители аргументов соответствуют локали (по умолчанию используется запятая). Openpyxl не вычисляет формулы, поэтому ошибки в них видны только при открытии файла.

**Проблемы с датами:** При отладке дат важно помнить об отличиях: Excel считает 1900 год високосным. Openpyxl обычно корректно это обрабатывает, но при ручных конвертациях можно наткнуться на смещение на день. Лучше всегда полагаться на автоматическое преобразование через `datetime`.

**Проблемы с памятью:** Если скрипт падает с `MemoryError` при работе с большим файлом, это сигнал к использованию режимов `read_only` или `write_only`. Отладка таких ситуаций требует профилирования памяти.

## 10. Композиция и организация результата

Это про то, как из простых ячеек и строк собрать сложный, профессионально оформленный Excel-документ.

**Композиция через множественные листы:** Организация результата часто начинается с распределения данных по разным листам. Вы можете создать книгу с листами "Сводка", "Детальные данные", "Графики", что делает информацию структурированной и удобной для навигации.

**Композиция стилей (Named Styles):** Вместо того чтобы определять шрифт и заливку для каждой ячейки отдельно, можно создать именованный стиль (`NamedStyle`) и применять его группам ячеек. Это аналог CSS для Excel, позволяющий организовать единообразное оформление и менять его централизованно.

```python
from openpyxl.styles import NamedStyle, Font, PatternFill
highlight = NamedStyle(name="highlight")
highlight.font = Font(bold=True)
highlight.fill = PatternFill(...)
wb.add_named_style(highlight)

for cell in ws['A1:A10']:
    cell.style = "highlight"
```

**Композиция через диаграммы и изображения:** Итоговый результат может включать в себя не только данные, но и визуальные элементы. Openpyxl позволяет создавать диаграммы на основе диапазонов данных и вставлять изображения, привязывая их к определенным ячейкам. Это превращает набор цифр в полноценный отчет.

**Организация печати и просмотра:** Библиотека позволяет управлять параметрами страницы: устанавливать область печати, повторять заголовки строк на каждой странице, добавлять верхние и нижние колонтитулы. Это финальный этап организации результата для передачи или презентации.

# Модуль 3. Практическое применение

## 11. Формулировка практического задания

**Контекст:** В любой организации, от малого бизнеса до корпораций, Excel является основным инструментом для финансовой отчетности, ведения списков, учета и планирования. Ручное создание таких отчетов отнимает часы, подвержено ошибкам и требует рутинной работы по форматированию. Openpyxl позволяет полностью автоматизировать этот процесс, генерируя готовые к использованию файлы Excel программно.

**Прикладная задача:** Разработать скрипт "Автоматизированный генератор прайс-листов и отчетов", который демонстрирует возможности Openpyxl для:
1.  Создания новой рабочей книги с несколькими листами.
2.  Записи данных из различных источников (списки, словари).
3.  Применения профессионального форматирования (стили, выравнивание, границы).
4.  Добавления формул для автоматического подсчета итогов.
5.  Настройки внешнего вида столбцов и строк.

```python
from openpyxl import Workbook
from openpyxl.styles import Font, Alignment, PatternFill, Border, Side
from openpyxl.utils import get_column_letter

# Исходные данные (обычно приходят из БД или API)
products = [
    {"название": "Системный блок", "количество": 5, "цена": 45000.00},
    {"название": "Монитор 24\"", "количество": 3, "цена": 12000.00},
    {"название": "Клавиатура", "количество": 10, "цена": 1500.00},
    {"название": "Мышь", "количество": 15, "цена": 800.00},
]

# 1. Создаем книгу и лист
wb = Workbook()
ws = wb.active
ws.title = "Прайс-лист"

# 2. Заголовки
headers = ["№", "Товар", "Кол-во", "Цена (₽)", "Сумма (₽)"]
ws.append(headers)

# 3. Стиль для заголовков
header_font = Font(bold=True, color="FFFFFF")
header_fill = PatternFill(start_color="1E90FF", end_color="1E90FF", fill_type="solid")
header_alignment = Alignment(horizontal="center", vertical="center")
thin_border = Border(left=Side(style='thin'), right=Side(style='thin'),
                     top=Side(style='thin'), bottom=Side(style='thin'))

for col_num, cell in enumerate(ws[1], 1):
    cell.font = header_font
    cell.fill = header_fill
    cell.alignment = header_alignment
    cell.border = thin_border

# 4. Заполняем данными и формулами
for idx, prod in enumerate(products, start=1):
    row = [
        idx,
        prod["название"],
        prod["количество"],
        prod["цена"],
        f"=C{idx+1}*D{idx+1}"  # Формула суммы для текущей строки
    ]
    ws.append(row)

# 5. Формула итого внизу
last_row = ws.max_row
ws[f'E{last_row+1}'] = f"=SUM(E2:E{last_row})"
ws[f'C{last_row+1}'] = "ИТОГО:"

# 6. Форматирование данных
for row in ws.iter_rows(min_row=2, max_row=last_row):
    for cell in row:
        cell.border = thin_border
        if cell.column in [3, 4, 5]:  # Числовые столбцы
            cell.alignment = Alignment(horizontal="right")
        else:
            cell.alignment = Alignment(horizontal="left")

# 7. Автоподбор ширины столбцов
for col in ws.columns:
    max_length = 0
    col_letter = get_column_letter(col[0].column)
    for cell in col:
        try:
            if cell.value:
                max_length = max(max_length, len(str(cell.value)))
        except:
            pass
    adjusted_width = min(max_length + 2, 50) # Не даем стать слишком широким
    ws.column_dimensions[col_letter].width = adjusted_width

# 8. Сохраняем файл
wb.save("сгенерированный_прайс-лист.xlsx")
print("Файл 'сгенерированный_прайс-лист.xlsx' успешно создан!")
```

## 12. Архитектура решения

Решение строится как модульное приложение, демонстрирующее различные сценарии работы с Excel.

**Структура проекта:**

1.  **Модели данных (Data Models):** Простые классы или словари, представляющие сущности (Продукт, Сотрудник, Заказ).
2.  **Генераторы отчетов (Report Generators):** Модули, отвечающие за создание специфических типов Excel-файлов.
3.  **Утилиты форматирования (Formatting Utilities):** Набор функций для применения стандартных стилей, настройки ширины и т.д.
4.  **Слой ввода/вывода (I/O Layer):** Функции для загрузки исходных данных (из CSV, JSON) и сохранения итоговых файлов.

```python
# --- models.py ---
class Product:
    def __init__(self, id, name, price, stock):

        self.name = name
        self.price = price

# --- formatters.py ---
from openpyxl.styles import Font, PatternFill, Alignment, Border, Side
from openpyxl.worksheet.worksheet import Worksheet

def apply_header_style(ws: Worksheet, row_idx: int = 1):
    """Применяет стандартный стиль к строке заголовка."""
    header_font = Font(bold=True, size=12, color="FFFFFF")
    header_fill = PatternFill(start_color="2F75B5", end_color="2F75B5", fill_type="solid")
    for cell in ws[row_idx]:
        cell.font = header_font
        cell.fill = header_fill
        cell.alignment = Alignment(horizontal="center")

def autofit_columns(ws: Worksheet):
    """Автоподбор ширины столбцов."""
    from openpyxl.utils import get_column_letter
    for col in ws.columns:
        max_length = 0
        col_letter = get_column_letter(col[0].column)
        for cell in col:
            if cell.value:
                max_length = max(max_length, len(str(cell.value)))
        ws.column_dimensions[col_letter].width = max_length + 2

# --- generators.py ---
from openpyxl import Workbook
import models
import formatters

def generate_inventory_report(products: list[models.Product], filename: str):
    wb = Workbook()
    ws = wb.active
    ws.title = "Inventory"

    # Заголовки
    headers = ["ID", "Product Name", "Price", "In Stock"]
    ws.append(headers)
    formatters.apply_header_style(ws)

    # Данные
    for p in products:
        ws.append([p.id, p.name, p.price, p.stock])

    # Формула для подсчета общей стоимости запасов

    ws[f'C{last_row+2}'] = "Total Value:"
    ws[f'D{last_row+2}'] = f"=SUMPRODUCT(C2:C{last_row}, D2:D{last_row})"

    formatters.autofit_columns(ws)
    wb.save(filename)
    print(f"Report saved as {filename}")

# --- main.py ---
```

## 14. Возникшие сложности

**Сложности работы с Openpyxl**

### 1. Понимание разницы между индексами строк/столбцов и координатами
Новички путают 0-индексацию в Python и 1-индексацию в Excel. В Openpyxl строки и столбцы нумеруются с 1.
```python
# Правильно: обращение к первой строке, первому столбцу
cell = ws.cell(row=1, column=1)
# Неправильно: row=0 вызовет ошибку
```

### 2. Работа с датами и часовыми поясами
Excel имеет свою систему дат. Openpyxl использует `datetime`, но при очень старых датах могут быть расхождения. Также библиотека не обрабатывает часовые пояса, что может привести к смещению при открытии в Excel.
```python
from datetime import datetime, timezone
# Рекомендуется передавать "наивные" datetime объекты (без timezone)
ws['A1'] = datetime.now().replace(tzinfo=None)
```

### 3. Производительность и память при больших файлах
Самый частый "камень преткновения". Загрузка файла на 100 Мб стандартным `load_workbook` может занять 10-15 секунд и потребовать более 1 Гб RAM.
```python
# Решение для больших файлов на чтение
wb = load_workbook('big.xlsx', read_only=True)
# Решение для записи огромных объемов
wb = Workbook(write_only=True)
```

### 4. Повреждение файла при сохранении
Если не вызвать `wb.save()`, все изменения будут потеряны. Также повреждение может возникнуть, если в процессе работы произошла ошибка и скрипт завершился аварийно. Рекомендуется использовать конструкцию `try...finally` для гарантированного сохранения или закрытия файла.

**Ограничения проекта**

### 1. Отсутствие поддержки старого формата .xls
Openpyxl **не работает** с бинарными файлами Excel 97-2003 (`.xls`). Попытка открыть такой файл вызовет `InvalidFileException` .

### 2. Ограниченная поддержка макросов (VBA)
Хотя Openpyxl может открывать и сохранять файлы с макросами (`.xlsm`), она не умеет их читать, изменять или создавать. При сохранении файла с макросами нужно указывать `keep_vba=True` .

### 3. Невозможность запуска Excel и выполнения вычислений
Openpyxl — это библиотека для работы с файлами, а не с приложением Excel. Она не может запустить Excel, чтобы пересчитать формулы или выполнить макрос. Для получения результатов формул файл нужно открыть в Excel вручную, либо использовать флаг `data_only` при загрузке файла, который уже был вычислен.

### 4. Ограниченная поддержка некоторых сложных диаграмм
Хотя базовая функциональность диаграмм есть, очень сложные или нестандартные типы диаграмм могут не поддерживаться или приводить к повреждению файла.

## 15. Итоговая оценка инструмента

Openpyxl — это **промышленный стандарт** для работы с современными Excel-файлами в Python. Она предоставляет наиболее полный и гибкий набор инструментов для автоматизации задач, связанных с таблицами.

**Ключевые преимущества:**
- **Полнота функционала:** Позволяет делать с Excel-файлом практически всё: от чтения данных до создания сложных стилизованных отчетов с диаграммами.
- **Объектная модель:** Интуитивно понятный API, отражающий структуру самой Excel-книги (книга -> лист -> ячейка).
- **Стабильность и зрелость:** Библиотека существует много лет, имеет большое сообщество и активно поддерживается, что гарантирует надежность .
- **Эффективность:** Наличие потоковых режимов (`read_only`/`write_only`) позволяет обрабатывать файлы, неподъемные для многих других инструментов.

**Когда использовать:**
- Автоматическая генерация любых Excel-отчетов (финансовых, статистических, для клиентов).
- Импорт данных из Excel в систему (парсинг загрузок от пользователей).
- Пакетная обработка множества Excel-файлов.
- Создание шаблонов и их заполнение данными.

**Сравнение с альтернативами:**
| Инструмент | Сильные стороны | Слабые стороны |
| :--- | :--- | :--- |
| **Openpyxl** | Полный контроль над форматом `.xlsx`, стили, диаграммы, формулы. | Не работает со старым `.xls`, требует памяти для больших файлов (без оптим. режимов). |
| **xlrd / xlwt** | Чтение старого `.xls` (xlrd). | Ограниченный функционал, нет поддержки `.xlsx`, устаревшие. |
| **pandas** | Мощный анализ данных, простота записи DataFrame в Excel. | Тяжеловесна, ограниченный контроль над форматированием (требует Openpyxl/XlsxWriter как движок). |
| **XlsxWriter** | Очень быстрая запись, отличная поддержка диаграмм. | Не умеет читать или изменять существующие файлы. |

**Вывод:**
Openpyxl — это незаменимый инструмент для Python-разработчика, сталкивающегося с задачами автоматизации отчетности или интеграции с Excel. Освоив его, вы сможете превратить ручную, трудоемкую работу по созданию таблиц в быстрый, надежный и воспроизводимый программный процесс.
